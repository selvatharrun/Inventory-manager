063	normal	ABSTRACT
065	normal	The problem addressed in this project is the difficulty operations teams face in making reliable inventory decisions under demand variability, supplier lead-time shifts, and incomplete context. Manual spreadsheet workflows are slow and often inconsistent across planners.
066	normal	The main aim of this project is to develop an Inventory Optimization AI Agent that analyzes SKU-level stock data and produces advisory recommendations for replenishment and risk mitigation.
067	normal	The project integrates data loading, metric computation, rule evaluation, hybrid knowledge graph enrichment, and batched LLM explanation generation. LangGraph orchestrates the workflow and FastMCP exposes independently testable tools.
068	normal	The dataset used in implementation consists of structured inventory records with 25 synthetic SKUs across multiple categories and demand profiles.
069	normal	The system is evaluated on decision quality signals, output completeness, fallback reliability, and end-to-end latency under CPU-only constraints.

090	normal	1.1 Background of the Study
091	normal	Data inventory recommendation is a fundamental component of data analytics, enabling users to interpret complex inventory inventory datasets through graphical representations. With the exponential growth of data across industries, there is an increasing need for tools that simplify the process of analyzing and visualizing data to drive informed decision-making. Effective inventory recommendation transforms raw numbers into intuitive patterns, trends, and correlations that might otherwise go unnoticed in structured or unstructured formats.
092	normal	Traditional inventory recommendation tools such as Python libraries (Matplotlib, Seaborn) and specialized business intelligence platforms require users to possess significant programming knowledge or technical expertise. This creates a substantial barrier for non-technical users, such as business managers and healthcare professionals, who wish to explore data insights independently without relying on data science teams. The learning curve associated with syntax and data preprocessing often results in underutilized data assets.
093	normal	Recent advancements in artificial intelligence, particularly in natural language processing (NLP), have enabled systems to understand and interpret human language effectively. Large Language Models (LLMs) have shown significant capabilities in translating natural language inventory requests into executable code or structured outputs, bridging the gap between human intent and technical execution. These models can identify entities, intent, and context within a user's request to automate complex analytical workflows.
094	normal	This project leverages these advancements to create an Inventory Optimization AI Agent capable of generating inventory optimizations. By integrating conversational AI with robust inventory recommendation libraries, the system aims to democratize access to data analytics tools, allowing users to upload inventory inventory datasets and interact through simple natural language to produce high-quality, interactive recommendations automatically. This shift toward "natural language interfaces for data" represents a significant move toward making data-driven insights accessible to everyone, regardless of their technical background.

180	Heading 2	2.2 Related Work
181	normal	Data inventory recommendation has traditionally been performed using tools such as Tableau, Power BI, and programming libraries like Matplotlib and Seaborn. These tools require users to have technical expertise or prior experience in handling inventory inventory datasets.
182	normal	Monika (2007) developed early inventory recommendation techniques focusing on static recommendation generation, which required manual input and configuration. Later, Ramakrishna (2009) and Bhaskar (2010) extended these approaches by integrating automation features, but they still required coding knowledge.
183	normal	With the advancement of machine learning, researchers began exploring automated inventory recommendation systems. Rao and Ram (2011) proposed a semi-automated system that suggested recommendations based on inventory dataset features. However, user interaction was limited and not conversational.
184	normal	The introduction of Natural Language Processing (NLP) significantly changed the landscape. Systems like NL4DV (Natural Language for Data Visualization) enabled users to generate recommendations using natural language inventory requests. These systems used rule-based and machine learning approaches to interpret inventory requests.
185	normal	More recently, large language models (LLMs) such as GPT and transformer-based architectures have enabled more flexible and accurate interpretation of user inventory requests. Singh and Robin (2008); Ram et al. (2009); and Prakash (2011) demonstrated how AI models can improve user interaction in data-driven systems.
186	normal	Modern AI-driven tools integrate conversational interfaces with inventory recommendation engines. These systems allow users to interact with data using simple language, reducing the need for technical expertise. However, many existing systems rely on cloud-based APIs, raising concerns about privacy and cost.
